OG images are generated server-side at `/og.*` routes using `@vercel/og` (Satori). They render at the standard **1200×630** dimensions expected by Twitter, Facebook, Slack, Discord, and other unfurlers.

## Routes

| Route | Description |
|-------|-------------|
| `/og` | Default app card with name and tagline |
| `/og/user/:handle` | Profile card with avatar, name, bio, follower counts |
| `/og/post/:id` | Post card with text, author, engagement stats |
| `/og/article/:handle/:slug` | Article card with title, subtitle, author, reading time |

## Architecture

All routes share these building blocks from `src/lib/og-image.tsx`:

- **`OgFrame`** — the 1200×630 canvas with a seeded gradient background (blue-to-violet band), a branded eyebrow label, and the app wordmark in the bottom-left
- **`OgAvatar`** — renders a remote avatar (transcoded to PNG via sharp) or falls back to a gradient initial chip
- **`OgStats`** — horizontal row of compact stats (e.g. "1.2k followers · 340 following")
- **`getOgFonts()`** — fetches Inter font weights from Google Fonts (woff format for Satori compatibility)
- **`loadOgImage(src)`** — fetches and transcodes remote images to PNG data URLs with a 4s timeout

## Styling

The gradient background is deterministic — seeded from the content (post ID, handle, etc.) so each card gets a unique-but-consistent hue in the blue-to-violet range.

Font sizes adapt to content length. For example, post text uses 60px for short posts and 42px for longer ones.

## Caching

All OG images are cached with `Cache-Control: public, max-age=3600, s-maxage=3600, stale-while-revalidate=86400` (1hr fresh, 24hr stale-while-revalidate).
